# GitHub Overview

## What is GitHub?
**GitHub** is a cloud-based platform that hosts Git repositories. It provides a web-based interface and additional collaboration tools on top of the core Git functionality.

While **Git** is the engine that tracks changes, **GitHub** is the garage and the social network where developers store their code and collaborate with others.

## Why do we need it?

### 1. Centralized Hosting
GitHub acts as the "Single Source of Truth" for our project. It allows us to keep our code in a central, secure location that is accessible from anywhere in the world.

### 2. Pull Requests (PRs)
The most powerful feature of GitHub. A Pull Request is a way to propose changes to the codebase. It allows other developers to:
- Review the code line-by-line.
- Discuss implementation details.
- Run automated tests before the code is merged into `master`.

### 3. Issue Tracking
GitHub provides a way to track bugs, feature requests, and tasks. This helps us organize what needs to be done and who is working on what.

### 4. Collaboration Tools
- **Code Review**: Inline comments and approvals.
- **Actions**: Automated workflows (CI/CD) that run tests or deploy code automatically.
- **GitHub CLI (`gh`)**: A tool that allows us to interact with GitHub directly from our terminal.

## Global Adoption
GitHub is the world's largest development platform:
- **Users**: Over 100 million developers.
- **Repositories**: Hosting over 330 million repositories.
- **Enterprise**: Used by over 90% of Fortune 100 companies.
