# PR-11: feat: infrastructure for automatic Jenkins jobs per PR

## Purpose
This Pull Request migrates our documentation and strategy towards Multibranch Pipelines. This new approach automatically creates Jenkins jobs for every Pull Request.

## Reviewer Reading Guide
1.  **`docs/infrastructure/jenkins.md`**: Review the newly added section detailing the benefits and configuration of Multibranch Pipelines.

## Key Changes
- **Multibranch Pipeline Migration**:
    - Transitioned from a single Pipeline job to a **Multibranch Pipeline** (`enterprise-multibranch`) that automatically manages isolated jobs for all branches and PRs.
    - **Final Stable Source**: Switched from "GitHub" source to a standard **Git** source with "Discover branches" traits. This reliably bypasses GitHub API rate limits (60/hr anonymous) while still using PAT authentication for private access.
- **Concurrency Control**:
    - Configured Jenkins to use exactly **1 executor** on the built-in node. This disables parallelism, ensuring only one build runs at a time for better stability in the local Docker environment.
- **Security & Infrastructure**:
    - Securely injected and verified the `github-token` credential.
    - Recovered the Jenkins instance from experimental configuration boot-loops into a clean, stable state.
- **Documentation**:
    - Fully updated `jenkins.md`, `README.md`, and the Tech Learning Center index to reflect the new Multibranch architecture and single-build constraint.
