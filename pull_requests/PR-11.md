# PR-11: feat: infrastructure for automatic Jenkins jobs per PR

## Purpose
This Pull Request migrates our documentation and strategy towards Multibranch Pipelines. This new approach automatically creates Jenkins jobs for every Pull Request.

## Reviewer Reading Guide
1.  **`docs/infrastructure/jenkins.md`**: Review the newly added section detailing the benefits and configuration of Multibranch Pipelines.

## Key Changes
- **Automated Job Creation**:
    - Implemented a series of Groovy initialization scripts to automatically create and configure the `enterprise-multibranch` job.
    - **Secure Token Injection**: Securely retrieved the GitHub token from the local `gh` CLI and injected it into Jenkins as a "Secret Text" credential.
    - **API Rate Limit Fix**: Configured Jenkins to use the `ThrottleOnOver` strategy for the GitHub API to bypass proactive throttling and unblock branch discovery.
    - Verified successful job creation, webhook activation, and branch indexing via Jenkins logs.
- **Comprehensive Documentation**:
    - Added "Advanced: Multibranch Pipelines (Automatic PR Jobs)" section to `jenkins.md`.
    - Included a "Troubleshooting & Maintenance" section for managing API rate limits and updating tokens.
